# Issue tracker

Local markdown. Issues live as files under `.scratch/issues/` in this repo.

## Convention

- One directory per feature or bug: `.scratch/issues/<slug>/`
- Each directory contains a `README.md` with the issue body
- Status is tracked via a `status:` frontmatter field: `open`, `in-progress`, `closed`
- Agents create issues with `status: open` and never close them without human confirmation
